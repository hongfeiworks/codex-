# comment-doc-zh

请针对下面的代码或改动补充必要的中文注释与中文文档：

`$ARGUMENTS`

## 目标

- 补足真正有价值的中文注释
- 让后续维护者能快速理解关键流程、约束和边界
- 不制造废话注释和模板化文档

## 注释规则

1. 注释与文档统一使用简体中文。
2. 只在这些场景补注释：
   - 核心流程
   - 业务约束
   - 边界条件
   - 性能取舍
   - 非直觉实现
   - 容易误用的接口或参数
3. 不要写这种低价值注释：
   - 解释显而易见的赋值、判断、遍历
   - 把代码原样翻译成中文
   - 已经过时或未来容易失真的说明
4. 若改动影响使用方式、配置方式或迁移步骤，补一段简洁的中文文档。
5. 文档要贴合当前代码和当前仓库，不要写泛泛而谈的背景介绍。

## 输出要求

1. 先判断哪些地方真的需要注释或文档，哪些地方不需要。
2. 直接完成修改。
3. 完成后说明：
   - 新增了哪些中文注释或文档
   - 为什么这些地方值得写
   - 哪些地方刻意没写，以及原因
